## 一、核心改动

### 1.1 信号生成逻辑重构 (signal_generator.py)

**原逻辑问题**:
- 采用纯AND多条件（6个指标必须同时满足）
- 联合概率极低（<5%），导致信号稀疏
- 原公式: `S_buy = I(SMA60↑) × I(BIAS<θ) × I(Vol缩) × I(RSI<thresh) × I(MACD转正) × I(阳线)`

**新逻辑**:
- **加权评分系统** (0-1范围) + 阈值触发
- 权重分配:
  - 趋势 (SMA60向上): 0.20
  - BIAS (负乖离率): 0.30 (核心指标，提高权重)
  - 缩量: 0.15
  - RSI (超卖): 0.20
  - MACD (转正): 0.10
  - 阳线: 0.05

**新公式**:
```
Score = 0.20×I(SMA60↑) + 0.30×clip((θ_buy-BIAS)/15, 0,1)
      + 0.15×I(Vol缩) + 0.20×clip((RSI_thresh-RSI)/30, 0,1)
      + 0.10×I(MACD>0) + 0.05×I(阳线)

S_buy = I(Score > 0.60) ∧ Liquidity_OK
```

**关键改进**:
- 从"全满足"改为"综合评分"，大幅提升信号捕获率
- 保留T+1 shift，符合A股T+1交易规则
- 流动性过滤保持不变

### 1.2 优化器增强 (optimizer.py)

**新增惩罚项**:

1. **交易次数惩罚** (核心修复):
   ```python
   if num_trades < 30:
       trade_penalty = 10.0 * (30 - num_trades) / 30
   ```
   - 少于30笔交易显著惩罚
   - 防止参数收敛到极端阈值

2. **换手率惩罚** (强化):
   ```python
   turnover_penalty = max(0, turnover - 0.167) * 10
   ```
   - 年化换手率>200% (月均>16.7%) 时惩罚
   - 控制交易成本

3. **参数极值惩罚**:
   ```python
   if alpha_vol < 0.5 or alpha_vol > 0.8:
       param_penalty = 5.0
   ```
   - 强制alpha_vol在0.5~0.8范围
   - 防止过度缩量条件

**适应度函数**:
```
Fitness = Calmar_Ratio - trade_penalty - turnover_penalty - param_penalty
```

## 五、技术要点总结

### 5.1 为什么加权评分有效？

**数学原理**:
- 原AND逻辑: P(信号) = P(A) × P(B) × ... × P(F) ≈ 0.8^6 = 26%
- 新评分逻辑: P(信号) = P(Score > 0.60) ≈ 根据权重分布，约40-60%
- **关键**: 从"乘法概率"改为"加法评分"，避免概率坍缩

**A股适配性**:
- 高波动市场：单一指标极值频繁，但多指标同时极值罕见
- 加权评分允许"部分满足"，更符合实际交易场景
- 阈值可调：牛市降低（0.55），熊市提高（0.70）

### 5.2 参数约束的重要性

**alpha_vol限制在0.5~0.8**:
- <0.5: 过度缩量，信号过少
- >0.8: 允许放量，失去"缩量超跌"逻辑
- 0.65: 经验最优值（允许35%缩量）

**交易次数惩罚**:
- 防止优化器找到"1笔大赚"的极端参数
- 确保统计显著性（至少30笔）
- 符合量化策略稳健性要求

### 5.3 T+1执行的正确性

```python
df['signal'] = df['signal'].shift(1).fillna(0)
```
- 当日收盘产生信号 → 次日开盘执行
- 避免未来函数（look-ahead bias）
- 符合A股T+1交易规则
