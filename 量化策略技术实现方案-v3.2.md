## 问题诊断
基于仓库v3.1的result_v3.1/文件夹（backtest_report_v3.1.md：年化收益~18-22%、Sharpe~1.4-1.6、换手率<10%/月、胜率>55%；tuned_run_v3.1.log：优化日志显示num_trades~20-25），交易次数偏低的根因在于信号生成与优化的概率乘积效应。详细拆解如下：

1. **阈值与过滤仍偏保守（主因）**  
   v3.1虽放宽阈值（e.g., BIAS20阈值3-5%、RSI<35、α_vol 0.5-0.8），但6个AND条件（SMA60向上 + BIAS< -4% + 缩量 + RSI<35 + MACD转正 + 阳线）的联合概率在A股高波动+T+1环境下仍低（日频命中率~2-5%）。流动性过滤（市值>50亿、成交>5000万）进一步压缩信号空间。报告中2025-2026测试集仅捕捉到天孚通信（300394.SZ）的~10个买点，但全池平均每股年信号<2个，导致总交易稀疏。  
   原信号生成公式（LaTeX）：  
   $$
   S_{\text{buy},t} = \prod_{i=1}^{6} \mathbb{I}(\text{Condition}_i)_t \land \text{Liquidity OK}_t
   $$

2. **优化器偏好低频参数（副作用）**  
   遗传算法（GA）+Walk-Forward优化（WFO）的目标函数（Calmar比率 - 换手惩罚）虽罚高换手（>15%/月 ×10），但未罚低交易，导致参数收敛到保守阈值（e.g., α_vol~0.6偏高）。日志显示种群/代数（50/50）不足以探索高频参数空间，优先最小化DD而非均衡信号量。

3. **微观与数据层放大（次要）**  
   stock_filter.py的每日筛选（ROE>15%、SMA60向上等）限制股票池至20-40只，结合T+1移位+停牌过滤（backtest_engine.py中valid=next day），使有效信号稀释。测试集周期虽扩展，但2025-2026窗口政策中性，缺失黑天鹅事件覆盖（如2018去杠杆、2020疫情），导致信号进一步低估。

**验证建议**：在signal_generator.py中添加debug打印，观察每日条件满足率和交易日志。

## 解决方案
采用3步优化框架：修改信号逻辑为加权分数、强化优化器惩罚、优化测试集选择与执行验证。保留原仓库结构，仅局部替换函数。方案强调政策权重窗口（预留>20%），以防纯历史回测失效。

### 步骤1：修改信号生成逻辑（signal_generator.py）
将纯AND替换为加权分数（0-1范围），阈值>0.65触发买入（较前方案放宽以提升次数）。权重分配参考Barra因子本土化：趋势/BIAS高权重（25%），辅助指标低权重。保留T+1移位、流动性过滤，并预留政策NLP接口（初始λ_policy=0.2）。

**修改后代码**（替换原generate_buy_signal函数）：
```python
# signal_generator.py - 优化版
import pandas as pd
import talib
import numpy as np

def generate_buy_signal(df: pd.DataFrame, params: dict) -> pd.Series:
    # 基础指标计算（同原仓库）
    df['sma60'] = talib.SMA(df['close'], 60)
    df['bias20'] = (df['close'] - talib.SMA(df['close'], 20)) / talib.SMA(df['close'], 20) * 100
    df['rsi14'] = talib.RSI(df['close'], 14)
    df['vol_sma10'] = talib.SMA(df['volume'], 10)
    macd, macdsignal, macdhist = talib.MACD(df['close'])
    df['macd_hist'] = macdhist
    df['is_yang'] = df['close'] > df['open']
    
    # 新增：信号强度分数（0-1）
    score = pd.Series(0.0, index=df.index)
    score += 0.25 * (df['sma60'] > df['sma60'].shift(1)).astype(int)          # 趋势权重25%
    score += 0.25 * np.clip((params['theta_buy'] - df['bias20']) / 8, 0, 1)    # BIAS超跌权重25%（分母放宽至8，提升敏感度）
    score += 0.15 * (df['volume'] < params['alpha_vol'] * df['vol_sma10']).astype(int)  # 缩量权重15%
    score += 0.15 * np.clip((params['rsi_thresh'] - df['rsi14']) / 15, 0, 1)   # RSI权重15%（分母放宽至15）
    score += 0.10 * (df['macd_hist'] > 0).astype(int)                         # MACD权重10%
    score += 0.10 * df['is_yang'].astype(int)                                 # 阳线权重10%
    
    # 预留政策权重窗口（未来接入BERT NLP）
    policy_score = get_policy_score(df['date'])  # 自定义函数，初始0.2权重
    score += 0.20 * np.clip(policy_score, 0, 1)  # 政策驱动加成
    
    # 流动性过滤（放宽成交阈值至>3000万以提升次数）
    df['liquidity_ok'] = (df['amount'] > 3e7) & (df['circulating_market_cap'] > 50e8)
    
    buy_signal = (score > 0.65) & df['liquidity_ok']
    return buy_signal.shift(1)  # T+1执行
````

**新买入公式（LaTeX）**：
$$
\text{Score}*t = 0.25 \cdot \mathbb{I}*{\text{SMA60 up}} + 0.25 \cdot \text{clip}\left(\frac{\theta_{\text{buy}} - \text{BIAS20}*t}{8},0,1\right) + 0.15 \cdot \mathbb{I}*{\text{Vol shrink}} + 0.15 \cdot \text{clip}\left(\frac{\text{rsi_thresh} - \text{RSI14}*t}{15},0,1\right) + 0.10 \cdot \mathbb{I}*{\text{MACD turn}} + 0.10 \cdot \mathbb{I}_{\text{Yang line}} + 0.20 \cdot \text{clip}(\text{NLP_policy}*t, 0, 1)
$$
$$
S*{\text{buy},t+1} = \mathbb{I}(\text{Score}_t > 0.65) \land \text{Liquidity OK}
$$

**参数建议**：θ_buy=-5%~-8%、rsi_thresh=30-40、α_vol=0.5-0.7（遗传算法自动优化）。
